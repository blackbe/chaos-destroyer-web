# 2026-02-18 — iMessage Gateway Permission Issue

## Problem
After reboot, iMessage wasn't arriving in OpenClaw. `imsg` CLI worked fine from terminal, but gateway failed with:
```
imsg rpc exited (code 1)
permissionDenied(path: "/Users/benblack/Library/Messages/chat.db", underlying: authorization denied (code: 23))
```

## Root Cause
The **OpenClaw gateway daemon** (running as LaunchAgent) spawns `imsg watch` via Node.js. While `imsg` CLI has Full Disk Access when run from terminal, the **Node.js process** running the gateway doesn't have FDA permissions — so it can't read Messages.db.

## Solution
Add Node.js binary to Full Disk Access:
1. System Preferences → Privacy & Security → Full Disk Access
2. Find Node path: `which node`
3. Add that path to FDA
4. Restart gateway: `openclaw gateway restart`

- ✅ `imsg watch --chat-id 84 --json` — streams messages correctly
- ✅ Manual send/receive test — message appeared in watch
- ✗ Gateway logs showed repeated permission denials trying to spawn `imsg watch`
